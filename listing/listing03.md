Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
	"fmt"
	"os"
)

func Foo() error {
	var err *os.PathError = nil
	return err
}

func main() {
	err := Foo()
	fmt.Println(err)
	fmt.Println(err == nil)
}
```

Ответ:
```
Из Foo мы возвращаем интерфейсный тип error

Вывод будет
<nil>
false

Интерфейс равен nil, только если и тип, и значение равны nil.
В функции Foo мы возвращаем ошибку у которой есть тип, и как следствие этот тип будет определен для интерфейса error
если дополнить код таким образом
fmt.Printf("%T, %v \n", err, err), то мы увидим в выводе что тип у err - *fs.PathError, а значение - <nil>

Интерфейс с методами:
type iface struct { // 16 bytes on a 64bit arch
    tab  *itab
    data unsafe.Pointer
}
Интерфейс представляет собой структуру, которая содержит 2 указателя:

iface - это структура, которая используется для представления интерфейсов, имеющих методы.
Она содержит два поля: tab, которое указывает на таблицу методов интерфейса, 
и data, которое является указателем на реальные данные.
Поскольку интерфейсы могут содержать только указатели, любое конкретное значение, которое мы переносим в интерфейс, должно иметь адрес.

Определяет набор методов, которые должны быть реализованы объектом, чтобы считаться совместимым с этим интерфейсом.
Позволяет объектам разных типов взаимодействовать друг с другом через общий интерфейс, не зная о конкретной реализации.
Обеспечивает полиморфизм и гибкость кода, позволяя работать с различными типами данных через общий интерфейс.

Пустой интерфейс (interface{}):
type eface struct {
	_type *_type
	data  unsafe.Pointer
}

_type, указывает на реальный тип данных, 
data, указатель на реальные данные.

Пустой интерфейс не имеет методов, что делает его универсальным контейнером для любых типов данных.
Может быть использован для хранения значений любого типа или для передачи значений через границы системы, где точный тип неизвестен.
Используется для обеспечения гибкости и возможности взаимодействия с объектами разных типов через общий интерфейс.
```
